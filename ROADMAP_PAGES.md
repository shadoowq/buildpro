# خطة صفحات BuildPro الجديدة — Landing Page & Auth Pages

## الهدف من هذا الملف
توثيق القرارات الخاصة بالصفحات العامة (قبل تسجيل الدخول) في BuildPro، عشان Claude Code يقدر يبني عليها مباشرة بدون الحاجة لإعادة شرح السياق.

---

## 1. الصفحة الرئيسية للشركة (Landing Page) — أولوية عالية

### لماذا هي ضرورية؟
حالياً أي زائر جديد (من واتساب، معرض بناء، إيميل تسويقي) بيوصله رابط الموقع ويلاقي صفحة login فاضية بدون أي سياق. ده بيضيع مجهود التسويق لأن الزائر مش بيفهم:
- مين BuildPro
- بيعمل ايه بالظبط
- ليه يسجل (كمقاول أو كمورد)

### المسار المقترح
`app/page.tsx` (الصفحة الرئيسية `/`) لازم تكون Landing Page عامة، مش redirect مباشر لصفحة login.

### الأقسام المطلوبة (بالترتيب)

1. **Hero Section**
   - عنوان رئيسي قوي: "اطلب تسعير مواد البناء من 300+ مورد في دقائق"
   - عنوان فرعي يشرح الفكرة في سطر واحد
   - زرين CTA: "سجل كمقاول" و"سجل كمورد" (يوديان لصفحة signup مع query param يحدد نوع الحساب مسبقاً، مثلاً `/signup?type=contractor`)

2. **إزاي بيشتغل (How It Works)**
   - 3 خطوات بسيطة مع أيقونات: اطلب التسعير → استقبل عروض من الموردين → قارن واختار أفضل عرض

3. **إحصائيات / أرقام**
   - عدد الموردين المسجلين، عدد المقاولين، عدد الطلبات المنفذة (إن وجدت بيانات حقيقية، أو placeholder قابل للتحديث لاحقاً)

4. **شعارات الموردين (Logo Wall)**
   - عرض لوجوهات أهم الموردين المسجلين لبناء المصداقية (مثال: RAK CERAMICS، Future Ceramics، إلخ من بيانات `users` الحالية)

5. **آراء العملاء (Testimonials)** — اختياري في المرحلة الأولى، يُضاف لاحقاً عند توفر شهادات حقيقية

6. **CTA ختامي**
   - دعوة أخيرة للتسجيل قبل الـ Footer

7. **Footer**
   - بيانات التواصل، روابط سريعة، الهوية البصرية (أسود + أبيض + أزرق بترولي `#0F4C75` / `#1B9AAA`)

### ملاحظات تصميم
- نفس الهوية البصرية المستخدمة في الـ Dashboard (خط Cairo، ألوان bp-navy/bp-teal)
- أسلوب احترافي مهني، شخصية قوية (راجع توجيهات الهوية البصرية في CLAUDE.md)
- الصفحة لازم تكون responsive لأن جزء كبير من الزيارات هتيجي من واتساب على الموبايل

## 2. صفحات Login / Signup — أولوية متوسطة

### الوضع الحالي
موجودة وشغالة وظيفياً (`app/login/page.tsx`, `app/signup/page.tsx`) لكن بدون هوية بصرية مميزة.

### المطلوب
- فصل بصري واضح عن باقي الموقع: شعار BuildPro بارز، نفس لوحة الألوان، عنصر بصري (illustration أو صورة) بجانب الفورم لإعطاء إحساس احترافي من أول لحظة
- الحفاظ على كل اللوجيك الوظيفي الموجود حالياً (لا حاجة لإعادة بناء منطق التسجيل)
- إضافة رابط واضح للرجوع للـ Landing Page (`/`)

---

## 3. صفحة رئيسية داخل الحساب (غير لوحة التحكم) — تم تأجيلها

### القرار: لا حاجة لها حالياً
السبب: المستخدم (مقاول أو مورد) بعد تسجيل الدخول عايز يشتغل فوراً — يشوف طلباته، يبعت طلب جديد، يراجع عروض الأسعار. صفحة وسيطة بين login والـ Dashboard هتبقى خطوة إضافية بدون فائدة حقيقية وممكن تتفهم كـ "شاشة ترحيب" بلا قيمة.

### البديل الحالي
الـ Dashboard (`app/dashboard/page.tsx`) يقوم بدور "الصفحة الرئيسية" للمستخدم فعلياً، لكن بمحتوى مفيد: إحصائيات حية، قائمة طلبات، تحديثات أخيرة، أفضل الموردين تفاعلاً.

### نقطة مستقبلية (ليست أولوية الآن)
"صفحة بروفايل عامة" لكل مورد — صفحة تعريفية قابلة للمشاركة كرابط تعرض اسم الشركة، التقييم، المنتجات. مفيدة تسويقياً للموردين لكن تُؤجَّل لمرحلة لاحقة من الـ roadmap.
